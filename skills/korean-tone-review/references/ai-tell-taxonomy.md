# AI 한글 티 분류 체계 (압축본)

> **출처**: [im-not-ai](https://github.com/epoko77-ai/im-not-ai) 의 `ai-tell-taxonomy.md` (MIT License, Copyright (c) 2026 epoko77-ai). 원본의 A~J 10개 대분류 × 세부 패턴 v2.0 체계를 본 스킬용으로 압축·재구성하여 사용.

LLM 한글 출력의 기계 냄새를 10개 축으로 체계화. 각 패턴마다 **정의 + 시그니처 예 + 심각도 + 처방** 제공.

## 심각도

- **S1 결정적** — 한 건만 나와도 AI 확신. 무조건 제거.
- **S2 강함** — 3회+ 누적 시 티 남. 밀도 기준 제거.
- **S3 약함** — 개별 무해, 다른 패턴과 겹치면 강화. 리듬 조정 수준.

---

## A. 번역투 (Translation-ese) — 가장 결정적 시그니처

영어/일본어 구문을 한국어 어순·조사로 무리하게 옮긴 흔적.

| ID | 패턴 | S | BAD → GOOD |
|---|---|---|---|
| A-1 | "~에 대하여/대해서" | S1 | "AI 규제**에 대해** 논의" → "AI 규제를 논의" |
| A-2 | "~를 통하여/통해" | S1 | "분석**을 통해** 인사이트" → "분석해 인사이트" |
| A-7 | "가지고 있다" | S1 | "강한 경쟁력을 **가지고 있다**" → "경쟁력이 강하다" |
| A-8 | 이중 피동 "~되어진다" | S1 | "판단**되어진다**" → "판단된다" |
| A-16 | 영어 대명사 직역 (그/그녀/그것) | S1 | "메리는 **그녀의** 어머니에게" → "메리는 어머니에게" |
| A-18 | 관계절 좌향 수식 과다 | S2 | "AI가 발전하는 가운데" 류의 긴 좌향 수식 |
| A-19 | 이중 조사 결합 | S2 | "에서의/로서의/만의" 등 부자연한 이중 |

## B. 영어 인용·용어 과다

| ID | 패턴 | S | 처방 |
|---|---|---|---|
| B-1 | 영어 단어 한국어 본문 혼용 | S2 | "default 로 ON 입니다" → "기본으로 ON" (단 `npm install` 같은 기술 식별자는 OK) |
| B-2 | 영어 표현 직역 단어 | S1 | "레시피"(recipes), "알아둘 것"(things to know), "한 번에 보기"(at a glance) — 자연스러운 한국어로 |
| B-3 | 괄호 병기 남발 | S3 | 첫 등장 1회만 영어 병기, 이후 한국어만 |

## C. 구조적 AI 패턴 (서식·레이아웃)

기계적 서식·도식적 구조.

| ID | 패턴 | S | BAD → GOOD |
|---|---|---|---|
| C-1 | 기계적 병렬 열거 | S1 | "첫째, ~. 둘째, ~. 셋째, ~." 전체 문단 → 일부 서술문으로 변주 |
| C-2 | Rhetorical 헤딩 (영어 직역) | S1 | `## 왜 X인가`, `## 무엇을 차단하는가`, `## 어떻게 동작하나`, `## 차단되면 어떻게 보이나` → 명사형 헤딩 (`## 배경`, `## 차단 대상`, `## 동작 방식`, `## 차단 예시`) |
| C-3 | 콜론 부제 헤딩 | S1 | `### 빠르게: X 스킬`, `### 단계 1: 설치` → 콜론 제거, 명사구로 (`### X 스킬로 편집`, `### 1. 설치`) |
| C-4 | Slash 구분 헤딩 | S1 | `### 시크릿 스캔 끄기 / 패턴 추가` → 슬래시 제거, 더 큰 개념으로 (`### 시크릿 스캔`) |
| C-5 | 이모지 남발 | S1 | "✅ 🚀 💡 ⚠️" — 에세이·리포트·README 에서 전량 제거 |
| C-9 | 숫자 괄호 인덱싱 "1) 2) 3)" | S2 | "1) X 2) Y 3) Z" → 1개는 서술문, 나머지는 "우선~·다음으로~"로 변주 |
| C-11 | 연결어미 뒤 쉼표 | S1 | "~지만**,** / ~고**,** / ~면서**,**" → 일괄 제거 (한국어 연결어미가 호흡 자동 생성) |

## D. AI 특유의 관용구 — 즉시 삭제 대상

인간 필자가 거의 안 쓰는 상투구.

| ID | 패턴 | S | 처방 |
|---|---|---|---|
| D-1 | 종결·요약류 | S1 | "결론적으로·따라서·요약하면·이를 통해" → 제거 또는 구체 명사·동사 |
| D-4 | Hype 어휘 | S1 | "혁신적·획기적·압도적·폭발적·파격적" → 구체 근거·수치로 대체 |
| D-5 | 메타 진입구 | S2 | "이는 ~을 의미한다", "이 관점에서 보면", "이 말은" → 본 서술로 직진 |
| D-6 | 완결 공식형 결말 | S2 | "~**해야 할 때입니다**", "~**시점입니다**" → 구체 동사 단언 |

## E. 리듬·문장 길이 균일성

| ID | 패턴 | S | 처방 |
|---|---|---|---|
| E-1 | 문장 길이 표준편차 낮음 | S2 | 모든 문장이 30~50자에 몰림 → 단문(10~15자) + 장문(80자+) 혼용 |
| E-4 | 단문 일변도 | S2 | "AI는 발전한다. 기업은 따라간다." → 연결어미·관형절로 복문화 |
| E-5 | 쉼표 분절 평균 길이 | S2 | AI 평균 8.56어절 vs 인간 4.35어절 → 7어절 초과 절은 분할 |
| E-7 | 청자 경어법 일관성 손실 | S2 | 같은 문서 "도와주시겠습니까? → 도와줘?" → 시작 시 단계 결정 후 유지 |

## F. 과도한 수식·중복

| ID | 패턴 | S | 처방 |
|---|---|---|---|
| F-1 | 정도부사 중독 | S2 | "매우·정말·진짜로·극히" → 대부분 삭제 또는 수치·사례 |

## G. 과도한 Hedging (완곡)

| ID | 패턴 | S | 처방 |
|---|---|---|---|
| G-1 | 추측·관측형 종결 | S2 | "~할 수 있을 것으로 보인다", "~인 것으로 판단된다" → 단언 가능하면 단언 |
| G-3 | 안전 균형 Lexicon | S2 | "양쪽 모두·두 가지 모두·신중하게·균형" 한 문서 4회 초과 → 한쪽 단언 또는 조건부 |

|---|---|---|---|

## I. 형식명사·의존명사 과다

| ID | 패턴 | S | 처방 |
|---|---|---|---|
| I-1 | "것이다" 종결 남발 | S2 | "~한 **것이다**" → 확정 서술 "~다" |
| I-2 | "점·바·수·데" 반복 | S2 | "주목할 **점**은·나아갈 **바**는" → 구체 명사·동사 |
| I-4 | "~해야 한다" 권고형 | S2 | "~**해야 한다**·~**해야 합니다**" 5회 초과 → 구체 동사 단언 |
| I-5 | "~하는 구조입니다 / ~식으로 설계되어 있어요" | S2 | "~한 줄로 적용하는 **구조입니다**" → "~한 줄로 적용합니다" |

## 종합 출력 형식

검수기는 발견 시:

```
findings:
  - id: <카테고리-ID>
    severity: S1|S2|S3
    span: "<원문에서 찾을 수 있는 짧은 quote>"
    suggestion: "<대안>"

ai_tell_density: <탐지건수 / 전체 글자 수>
severity_score: S1×5 + S2×2 + S3×0.5  # 0~100

verdict: accept | patch_needed | rewrite | hold_and_report
```

## 변경률 모니터링

- **5~30%**: 정상 범위
- **30% 초과**: 과윤문 의심 → 패치 범위 축소
- **5% 미만**: 저윤문 → S1 재확인
